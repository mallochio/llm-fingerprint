# Threat Model & Ethics

This document defines the security boundary, assumptions, and ethical scope of `llm-fingerprint`.

---

## 1. Threat Model

### In-Scope Threats
1. **Silent Model Swaps**: A provider or reseller silently downgrades or substitutes a claimed model (e.g. serving `gpt-4o-mini` while claiming `gpt-4o`).
2. **Opaque Router Auditing**: Measuring the backend distribution of auto-routing services (Cursor Auto, Devin Auto, RouterLLM) across multi-session samples.
3. **Reseller Fraud**: Detecting quantized, fine-tuned, or alternative open-weight variants served under proprietary model claims.

### Out-of-Scope / Weaknesses
1. **Adaptive Adversaries**: An adversary with full real-time knowledge of the exact battery prompts who overrides responses for those specific prompts (mitigation: holdout private batteries).
2. **Browser UI Scraping**: Scraping un-authenticated web UI chat windows (out of MVP scope).

---

## 2. Environment Tagging Rule

**API fingerprints $\neq$ Agent CLI fingerprints.**
Agent CLIs introduce system prompts, formatting wrappers, tools, and post-processing. Every fingerprint generated by `llm-fingerprint` is tagged with its `environment` (`openai-api`, `cursor-cli`, `devin-cli`). Verification compare routines enforce environment parity unless explicitly overridden by the user.

---

## 3. Responsible Usage & Safety

- **Workspace Isolation**: Always configure CLI adapters to run within an isolated empty directory (e.g. `/tmp/fingerprint-empty`).
- **No Exploitation**: `llm-fingerprint` is a diagnostic and auditing tool for endpoints the user is authorized to query. It does not perform security exploits or exfiltrate private code.
